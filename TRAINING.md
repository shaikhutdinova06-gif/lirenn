# Обучение ML модели (локально)

Render не предназначен для обучения моделей - это делается локально или в Colab.

## Локальное обучение

### 1. Подготовка датасета

Создай структуру папок:
```
dataset/
├── chernozem/
│   ├── image1.jpg
│   ├── image2.jpg
│   └── ...
├── podzolic/
│   ├── image1.jpg
│   └── ...
├── gray_forest/
├── gley/
├── chestnut/
└── tundra_gley/
```

Добавь изображения почв в соответствующие папки (минимум 50 фото на тип).

### 2. Запуск обучения

```bash
python backend/ml/train.py
```

Это создаст `backend/ml/model.pkl`.

### 3. Загрузка модели на GitHub

После обучения:
```bash
git add backend/ml/model.pkl
git commit -m "Add trained ML model"
git push
```

## Обучение в Google Colab

1. Открой https://colab.research.google.com
2. Создай новый ноутбук
3. Загрузи датасет
4. Скопируй код из `backend/ml/train.py`
5. Запусти обучение
6. Скачай `model.pkl`
7. Загрузи в `backend/ml/`

## Почему обучение не на Render?

- Render - для хостинга приложений, не для обучения
- Обучение требует много ресурсов (CPU/GPU)
- Бесплатный план Render имеет ограничения
- Обучение может занять много времени и прерваться

## Автообучение

Система автоматически сохраняет изображения для обучения когда:
- AI совпадает с картой
- Уверенность > 85%

Изображения сохраняются в `dataset_auto/` (не заливается на GitHub).
